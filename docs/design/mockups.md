# Mockups

## 文件目的
各頁面的低保真 wireframe，描述版位、資訊優先順序與互動位置。
本版本反映 **Direction C UI 改版**（決策於 2026-03-27）：首頁 / 相簿頁採淺色主題，照片詳情頁採暗色沉浸式主題。

---

## 設計原則（Direction C）

- **照片是主角**，UI chrome 必須退到背景
- 首頁與相簿頁：淺色（amber/stone 暖白），讓瀏覽輕鬆
- 照片詳情頁：暗色（#0c0c0c），最大化視覺沉浸
- 全站 UI 文字統一為英文
- 動畫：CSS transitions 為基礎，Framer Motion 用於照片頁進場與幻燈片切換

## 幻燈片模式（從照片頁開啟）

```text
+----------------------------------------------------------+  bg: #000
| ✕ Close                        3 / 24    ❙❙  ▶ Play     |
+----------------------------------------------------------+
|                                                          |
|     ‹           [FULL-SCREEN PHOTO]            ›        |
|                 Framer Motion crossfade                  |
|                                                          |
|          Tokyo Night · 2025-01-15 · Tokyo, JP           |
+----------------------------------------------------------+
| [⌄]  ┌───┐ ┌───┐ ┌[■]┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐  →   |  ← 縮圖帶（展開）
|      └───┘ └───┘ └───┘ └───┘ └───┘ └───┘ └───┘       |
+----------------------------------------------------------+

[縮圖帶最小化時：]
+----------------------------------------------------------+
| [⌃]  · · · ■ · · · · ·                                 |  ← dot indicator
+----------------------------------------------------------+
```

設計重點：
- 極黑背景，HUD 只有最基本資訊（進度 + 控制鈕）
- 底部縮圖帶：展開狀態顯示所有縮圖，橫向捲動；最小化後改為 dot indicator
- `[⌄]` / `[⌃]` toggle 按鈕控制縮圖帶展開/收合（CSS max-height transition）
- 當前縮圖自動捲動至可見範圍（scrollIntoView）
- 照片切換：Framer Motion opacity crossfade
