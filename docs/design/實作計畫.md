# 實作計畫

## 文件目的
這份文件是協作版的實作計畫，整理 Phase 1 到 Phase 3 的目標、目前實作現況、文件分工方式，以及接下來開發時應該優先對齊的地方。

規劃來源：
- `照片展示網站規劃_116679cc.plan.md`
- `.session/features.json`
- 目前 `src/`、`data/`、`prisma/` 內的實際程式結構

相關文件：
- `docs/design/workflow.md`：操作流程與頁面關係
- `docs/design/architecture.md`：程式架構、資料流、模組分層
- `docs/design/mockups.md`：各頁面的文字 wireframe / mockup

## 專案目標
建立一個以作品展示為核心的攝影網站，分成三個階段推進：

1. Phase 1：完成可公開瀏覽的照片展示與基本後台
2. Phase 2：補齊相簿、EXIF、幻燈片與瀏覽體驗
3. Phase 3：補上留言、按讚、管理與互動治理

## 現況摘要
目前專案已經不是純規劃階段，而是「有 MVP 骨架且已實作部分 Phase 2 / 3 能力」的狀態。

已存在的核心頁面：
- `/` 首頁照片牆
- `/albums/[slug]` 相簿頁
- `/photos/[source]/[id]` 單張照片頁
- `/admin/login` 後台登入頁
- `/admin` 後台管理頁

已存在的核心能力：
- 後台登入與 cookie session
- 照片上傳與多尺寸輸出
- 首頁照片牆與相簿入口
- 單張照片頁、上一張/下一張、幻燈片 viewer
- EXIF 顯示
- 留言新增與留言列表
- 按讚切換與按讚統計
- 相簿建立與編輯

目前資料層現況：
- 讀寫主來源是 `data/*.json`
- 圖片檔案目前落地在本機 uploads 流程
- `prisma/schema.prisma` 已存在，但尚未成為主要 runtime data source

## 協作建議
後續開發建議用這套文件分工：

1. `實作計畫.md`
   用來回答「做什麼、做到哪裡、下一步是什麼」。
2. `workflow.md`
   用來回答「使用者或管理者怎麼走到每個頁面與操作」。
3. `architecture.md`
   用來回答「程式要改哪一層、資料從哪裡進出、哪些模組可以重用」。
4. `mockups.md`
   用來回答「畫面應該長什麼樣、每區塊有哪些資訊與互動」。

## 文件維護原則
為了避免文件很快過期，建議遵守這幾條：

1. Phase 狀態以 `.session/features.json` 為準，這裡只做人類可讀摘要。
2. 若頁面、資料流、模組邊界有改動，優先更新 `workflow.md` 與 `architecture.md`。
3. 若 UI 結構有明顯改動，更新 `mockups.md`，不要只改畫面不改文件。
4. 每次 session 若有重大結構調整，在 `.session/progress.md` 補一筆摘要。

## UI 沉浸式改版計畫（Direction C，決策於 2026-03-27）

### 設計方向

採用 Direction C 混合主題：首頁 / 相簿頁為淺色，照片詳情頁為暗色沉浸式。
詳細規格見 `docs/design/mockups.md`、`docs/design/workflow.md`、`docs/design/architecture.md`。

### Feature Tickets（依實作順序）

| ID | 名稱 | 估計時間 | 依賴 |
|---|---|---|---|
| UI-THEME | 雙主題 CSS Token + Route Group Layout | 0.5 天 | P1-GALLERY |
| UI-NAV | Header 改版 + 全站英文化 | 0.5 天 | UI-THEME |
| UI-HERO | Hero Section 全出血 + 照片輪播 | 1.5 天 | UI-THEME, UI-NAV |
| UI-GALLERY | 照片牆視覺改版（移除卡片邊框） | 1 天 | UI-THEME |
| UI-PHOTOPAGE | 照片詳情頁暗色沉浸式改版 | 1.5 天 | UI-THEME |
| UI-SLIDESHOW | 幻燈片底部縮圖帶 + Framer Motion | 1.5 天 | UI-PHOTOPAGE |

**總估計**：約 6.5 天（含 Framer Motion 整合）

### 關鍵決策紀錄

| 決策 | 選擇 | 理由 |
|---|---|---|
| 色彩方向 | Direction C（混合） | 首頁好讀 + 照片頁最沉浸 |
| Hero 內容 | 照片輪播（client-side） | 照片是第一眼看到的，移除開發說明文字 |
| 動畫策略 | CSS 為主，Framer Motion 選擇性 | 80% 效果用 CSS 達到，避免過度依賴套件 |
| UI 語言 | 統一英文 | 攝影作品具國際性，i18n 另立 future work |
| 按讚設計 | 整合進留言區，移除獨立 Panel | 按讚是互動的一部分，不需要獨立佔空間 |
| 縮圖帶位置 | 只在幻燈片模式，底部水平帶 | 照片頁維持乾淨，幻燈片才需要導覽脈絡 |

## 建議近期優先項
比起再新增功能，近期更值得先整理的是：

1. 把 `features.json` 的狀態更新成符合現況
2. 明確標示哪些模組是「目前正式使用」，哪些是「為 Prisma / R2 遷移預留」
3. 先定義上傳、留言、按讚的驗證與錯誤處理規則
4. 補齊頁面層級的 acceptance criteria，避免規劃和程式各說各話
